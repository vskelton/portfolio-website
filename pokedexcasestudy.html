PokeDex Case Study

Project Overview

The PokeDex is a single page web application that allows users to view a list of Pokemon and access additional details for each.

My role for the project was to create a responsive web application that fetches data from the PokeAPI using HTML, CSS, and JavaScript. I also integrated Bootstrap to ensure a seamless user experience across various devices.

This project was completed over approximately one week as part of my coursework through CareerFoundry. As JavaScript was a new language for me, I focused on thoroughly understanding its concepts to build a functional and responsive application.

Problem & Opportunity

Before beginning this project, I identified an opportunity to apply my emerging web development skills to create an interactive and informative resource. The goal was to move beyond stattic data presentation and build a dynamic application where uesers could easily browse a large dataset (Pokemon) and retrieve specific details on demand. This also served as a practical challenge to master asynchronous JavaScript and API integration.

Project Goals

  • Load data from an external API(PokeAPI)
  • Display a comprehensive list of items
  • Enable users to view detailed information for an item upon interaction (e.g., clicking on a list item).
  • Ensure the application is fully responsive across different screen sizes.

Process and & Development Steps

1. Planning & Research

I began by researching the PokeAPI documentation to understand its structure and how to efficiently fetch the neccessary data.

2. Implementation

The core of the application involes using JavaScript's 'fetch' API with 'async/wait' to retrieve Pokemon data. I dynamically creaated list items for each Pokemon and appended them to the DOM. Event listeners were added to each item to trigger a modal that displayed additional details (height, types, abilities) fetched from a secondary API endpoint. Bootstrap's grid system and components were utilized to ensure the layout adapted smoothly to various viewport sizes.

3. Testing

Throughout development, I performed regular testing to ensure data was loaded correctly, interactions triggered as expected, and the UI remained consistent across desktop and mobile views. This involved checking API responses, debugging console errors, and manually resizing the browser window.

Challenges & Solutions

Developing the Pokedex web application presented a few different challenges that required some thoughtful solutions.

  • **Challenge: Asynchronous Data Handling:** Initially, managing asynchronous API calss and ensuring data was available before rendering presented a learning curve.
